Intelligent Enterprises APIs Overview

Enterprise Environment

Note: The Intelligent Enterprises APIs provide asset and location information, additional URI resources and data. Each API service is described below. Before using these APIs, start with the /assets or /locations endpoints described in the General APIs section.

The Enterprise Environment API provides indoor environment metadata collected continuously from sensors installed with intelligent lighting fixtures in a building, including:

  • Carbon Dioxide – measurement of carbon dioxide (CO₂) gas in parts-per-million (ppm), with a range from 0 ppm to 2000 ppm. This data can be used to monitor indoor air quality for occupant comfort, building HVAC system faults, and regulatory compliance.
  • Humidity – measurement of relative humidity, expressed as a percentage (%). This data can be used to monitor the amount of water vapor in the air for occupant comfort, building HVAC system faults, and regulatory compliance.
  • Illuminance – measurement of ambient light level in lux (lx), with a range from 0 lx to 3570 lx. This data can be used to maintain proper light levels for occupant comfort, reduce energy consumption when integrated with a daylight harvesting system, and monitor regulatory compliance.
  • Occupancy – measurement of occupancy, expressed as a Boolean value of either 0 or 1 to indicate occupied (1) or unoccupied (0). There are no units for this value. This data can be used to control lights, integrate with other building systems controls, and monitor usage of space.
  • Temperature – measurement of ambient temperature in degrees Celsius (°C), with a range from -273.15 °C to 327.67 °C. This data can be used to monitor facilities for occupant comfort, building HVAC system faults, and regulatory compliance.

This data is collected, stored with a time stamp, and presented as historical information for an individual asset (e.g. sensor). You can use this information to analyze patterns and trends to discover insights for enterprise energy efficiency and operations optimization.

Indoor Positioning

The Indoor Positioning API provides indoor position metadata collected from mobile devices running an application using the GE Indoor Positioning SDK. This application communicates with intelligent light fixtures enabled for indoor positioning applications in a building to determine the mobile device position.

Indoor positions are presented as Cartesian coordinates (X, Y, and Z), orientation angle, and duration in seconds. This data is collected, stored with a time stamp, and presented as historical information for an individual asset (e.g. mobile device). This information can be used to analyze the movement of visitor through a building space, including building zone dwell time and path analysis.
